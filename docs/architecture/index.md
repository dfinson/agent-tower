## Key Design Principles

- **Thin API routes** вҖ” Route handlers validate input, delegate to services, return results
- **Repository pattern** вҖ” All database access through repository classes
- **Event-driven** вҖ” All runtime activity flows through domain events
- **Adapter pattern** вҖ” Agent SDKs wrapped behind a common interface
- **Single source of truth** вҖ” Pydantic schemas for API, Zustand store for UI state

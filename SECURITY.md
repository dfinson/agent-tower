# Security Policy

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead, report vulnerabilities privately via our
[GitHub Security Advisories](https://github.com/codeplane-dev/codeplane/security/advisories)
page — click **Report a vulnerability** and provide:

- A description of the vulnerability
- Steps to reproduce
- The impact you've identified
- Any suggested fix (optional)

We aim to acknowledge reports within 48 hours and provide a fix or mitigation
plan within 7 days.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |
| Older   | ❌        |

Only the latest release receives security updates. We recommend always running
the most recent version.

## Security Documentation

See the [Security page](https://dfinson.github.io/codeplane/security/) in our
documentation for the full threat model, security features, and best practices.

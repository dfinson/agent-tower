0:00
Goal

The task was to add cost attribution — tracking which model calls cost how much per job, and making that data queryable via the analytics API.

0:03
Approach

After reading the existing analytics and telemetry modules, I planned to: create a new CostAttributionService with its own repository, feed it from RuntimeService after each telemetry span, and add an aggregation endpoint. This would require a new table and a parameter on setup_and_start().

0:05
Decision
Made cost_config nullable on setup_and_start() rather than required. Rationale: existing callers (including recover_on_startup()) don't have cost configuration. A required parameter would force all callers to construct a config object even when cost tracking is disabled. None means "no cost tracking for this job."
